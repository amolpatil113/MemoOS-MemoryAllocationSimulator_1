Algorithm 01

First Fit

Scans from the beginning. Places the process in the first hole large enough to fit. Fast — but causes early fragmentation.

✓ Fastest search
✗ High external fragmentation
Algorithm 02

Best Fit

Searches the entire list and picks the smallest hole that fits. Minimizes wasted space — but leaves tiny unusable fragments.

✓ Least wasted space
✗ Slowest — full scan every time
Algorithm 03

Worst Fit

Picks the largest available hole. Theory: leftover fragments are big enough to be useful. Reality: rarely works well.

✓ Large remainders
✗ Wastes large free blocks
Concept 04

Fragmentation

Over time, memory fills with scattered holes. Even if total free space is sufficient, no single hole is large enough — the system stalls.

Concept 05

Paging

Memory split into fixed-size frames. Processes split into equal pages. Pages map to any free frame — no external fragmentation.
